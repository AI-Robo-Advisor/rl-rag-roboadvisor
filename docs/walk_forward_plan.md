# Walk-Forward 분할 계획

> 관련 모듈: `src/rl/env.py` (이문정), `src/rl/backtest.py` (강유영)
>
> 데이터 기간: 2020-01-03 ~ 2025-03-28 (returns.parquet 기준, 총 1423 거래일)

---

## 왜 Walk-Forward인가

전체 기간을 단순 train/test로 분리하면 학습 시점보다 이전 정규화 파라미터(mean, std)가
미래 데이터에 누출되는 Look-ahead bias가 발생한다.
Walk-Forward는 학습 윈도우를 앞에서 뒤로 밀면서 각 윈도우에서 독립적으로 정규화하므로
bias를 방지하고, 시장 국면 변화(COVID 급락, 금리 인상기 등)에 대한 일반화 성능을 측정할 수 있다.

---

## 윈도우 정의

| Window | 학습(Train) | 검증(Test) | 거래일(추정) |
|--------|------------|-----------|-------------|
| W1 | 2020-01-01 ~ 2023-12-31 | 2024-01-01 ~ 2024-12-31 | 학습 ~1008일 / 검증 ~252일 |
| W2 | 2021-01-01 ~ 2024-12-31 | 2025-01-01 ~ 2025-03-28 | 학습 ~1008일 / 검증 ~59일 |

> W2 검증 기간은 데이터 수집 종료일(2025-03-28)까지만 유효하다.
> 이후 데이터를 추가 수집하면 W2 검증 기간을 2025-12-31까지 확장할 수 있다.

---

## 윈도우 설계 근거

- **학습 기간 4년**: PPO 에이전트가 충분한 시장 국면을 학습하려면 bull/bear 사이클이 최소 1회 이상 포함되어야 한다. 2020~2023은 COVID 급락(2020), 회복·성장기(2021), 금리 인상 하락기(2022), 반등기(2023)를 포함한다.
- **검증 기간 1년(W1)**: 단일 연도 검증으로 과적합 여부를 명확히 측정할 수 있다. 2024는 AI 반도체 랠리 국면으로 학습 기간과 성격이 다르다.
- **슬라이딩 스텝 1년**: 윈도우를 1년씩 밀어 W2 학습(2021~2024)이 금리 인상기를 중심으로 학습하도록 설계한다.

## 구현 가이드라인

```python
WINDOWS = [
    {"train_start": "2020-01-01", "train_end": "2023-12-31",
     "test_start":  "2024-01-01", "test_end":  "2024-12-31"},
    {"train_start": "2021-01-01", "train_end": "2024-12-31",
     "test_start":  "2025-01-01", "test_end":  "2025-03-28"},
]
```

각 윈도우에서:
1. 학습 기간 데이터만으로 Z-score 파라미터(mean, std) 계산
2. 계산된 파라미터로 검증 기간도 정규화 (학습 기간 통계 재사용)
3. PPO 에이전트를 학습 기간 데이터로 훈련
4. 검증 기간 데이터로 성과 측정
